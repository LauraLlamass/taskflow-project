# Backend API - Herramientas

En el desarrollo de APIs y aplicaciones backend se utilizan diferentes herramientas para facilitar la comunicación, pruebas, monitorización y documentación.

## Axios

Axios es un cliente HTTP simple basado en promesas para el navegador y node. js. Axios provee una librería fácil de usar en un paquete pequeño con una interfaz muy extensible.
Se utiliza principalmente en el frontend para comunicarse con el backend, enviando y recibiendo datos en formato JSON.

### ¿Por qué se usa?

- Simplifica las peticiones HTTP frente a `fetch`
- Maneja automáticamente respuestas JSON
- Permite interceptar peticiones y respuestas
- Mejora la legibilidad del código

## Postman

Postman es una plataforma de desarrollo y pruebas de APIs.
Permite enviar peticiones HTTP (GET, POST, PATCH, DELETE) y ver las respuestas del servidor.

### ¿Por qué se usa?

- Facilita probar endpoints de forma rápida
- Permite detectar errores en el backend
- Ayuda a documentar APIs
- Permite guardar colecciones de pruebas

## Sentry

Sentry es una plataforma de monitoreo de aplicaciones y seguimiento de errores en tiempo real (APM) diseñada para desarrolladores.
Identifica fallos, excepciones y problemas de rendimiento en entornos de producción, facilitando la detección rápida, el diagnóstico y la corrección de errores antes de que afecten a los usuarios finales.

### ¿Por qué se usa?

- Detecta errores automáticamente
- Permite ver trazas completas de errores
- Ayuda a mejorar la estabilidad de la aplicación
- Facilita el mantenimiento en producción

## Swagger

Swagger es un conjunto de herramientas de código abierto basado en la especificación OpenAPI que permite diseñar, construir, documentar y consumir APIs RESTful de manera estandarizada.
Facilita la creación de documentación interactiva (Swagger UI) que permite probar endpoints directamente desde el navegador, mejorando la colaboración y eficiencia en el desarrollo.

### ¿Por qué se usa?

- Genera documentación automática de la API
- Facilita la comunicación entre frontend y backend
- Permite probar endpoints sin herramientas externas
- Mejora la comprensión de la API

## Conclusión

Estas herramientas son fundamentales en el desarrollo moderno de aplicaciones:

- Axios → comunicación frontend-backend  
- Postman → pruebas de la API  
- Sentry → monitorización de errores  
- Swagger → documentación de la API  

Su uso mejora la calidad, mantenimiento y escalabilidad del software.
